\section{Executive Summary}

\textbf{Lithos} is an AI-powered research and diligence platform purpose-built for the critical minerals mining industry. We solve the fundamental problem that mining analysts face: extracting structured data from 400--500 page technical reports takes weeks per project, creating a massive bottleneck in the investment and operational decision-making process.

Our solution uses a fine-tuned LLM extraction pipeline to automatically pull NPV, IRR, CAPEX, reserves, resources, and AISC from NI 43-101 and JORC technical reports. We've indexed 15,000+ mining projects worldwide and provide instant AI-powered analysis that cuts analyst research time from \textbf{2 weeks to 2 minutes}.

\subsection{The Problem}

Mining is a \$2 trillion industry where every investment decision depends on reading and analyzing massive technical documents:

\begin{itemize}
    \item \textbf{400--500 page technical reports:} NI 43-101 (Canada), JORC (Australia), 43-101F1 reports
    \item \textbf{2 weeks per project:} Analysts manually extract NPV (page 201), reserves (page 87), CAPEX (page 143)
    \item \textbf{15,000+ global projects:} Impossible for analysts to track systematically
    \item \textbf{Scattered data sources:} SEDAR, SEC EDGAR, ASX, LSE, company websites
    \item \textbf{No centralized intelligence:} Bloomberg has commodity prices but not project-level technical data
\end{itemize}

\subsection{Our Solution}

\begin{enumerate}
    \item \textbf{Automated Extraction:} LLM-powered pipeline extracts structured data from technical reports automatically
    \item \textbf{Global Coverage:} 15,000+ projects across gold, copper, lithium, nickel, rare earths
    \item \textbf{AI Intelligence:} GPT-4-powered chat for instant risk analysis, project comparisons, due diligence
    \item \textbf{Real-Time Updates:} 24-hour processing of new filings, daily news refresh
    \item \textbf{Modern Interface:} Bloomberg-inspired UI with advanced filtering, bulk actions, export
\end{enumerate}

\subsection{Traction \& Validation}

\begin{itemize}
    \item \textbf{8 weeks} from concept to functional platform
    \item \textbf{15,000+} mining projects indexed worldwide
    \item \textbf{5 design partners} using daily (mining analysts, fund managers)
    \item \textbf{2 LOIs signed} from mid-market mining-focused funds
    \item \textbf{User feedback:} "Cuts research time from 2 weeks to 2 hours per project"
\end{itemize}

\subsection{Market Opportunity}

\begin{itemize}
    \item \textbf{\$2 trillion} global mining industry
    \item \textbf{10,000+} mining analysts, geologists, fund managers worldwide
    \item \textbf{\$500--\$2,000/month} willingness to pay for research automation tools
    \item \textbf{Adjacent markets:} Oil \& gas (\$3T), infrastructure (\$5T), real estate (\$10T)

\section{Product Overview}

\subsection{Core Value Proposition}

Lithos is the \textbf{only platform} that automatically extracts structured data from 400--500 page mining technical reports at scale, combines it with real-time news and regulatory filings, and provides AI-powered analysis in seconds.

\begin{itemize}
    \item \textbf{Automated Data Extraction:} LLM-powered pipeline extracts NPV, IRR, CAPEX, reserves, resources, AISC from NI 43-101 and JORC technical reports
    \item \textbf{Global Coverage:} 15,000+ projects spanning gold, copper, lithium, nickel, rare earths, and other critical minerals
    \item \textbf{Instant Intelligence:} AI chat interface for risk analysis, project comparisons, and due diligence queries in seconds
    \item \textbf{Real-Time News Integration:} Automated tracking of permitting updates, production results, commodity prices
    \item \textbf{Modern UX:} Bloomberg-inspired interface with advanced filtering, bulk actions, export capabilities
\end{itemize}

\subsection{Technical Architecture}

\subsubsection{Frontend Stack}
\begin{itemize}
    \item \textbf{Framework:} Next.js 15.2.4 with App Router
    \item \textbf{Language:} TypeScript (strict mode enabled)
    \item \textbf{UI Components:} Radix UI primitives + shadcn/ui component library
    \item \textbf{Styling:} Tailwind CSS with custom design system
    \item \textbf{Tables:} TanStack Table (formerly React Table) for advanced data grids
    \item \textbf{Charts:} Recharts for financial visualizations
    \item \textbf{Maps:} Mapbox GL JS for interactive geospatial views
\end{itemize}

\subsubsection{Backend Stack}
\begin{itemize}
    \item \textbf{Database:} Supabase (PostgreSQL) with Row Level Security (RLS)
    \item \textbf{Authentication:} Supabase Auth with email/password and OAuth providers
    \item \textbf{Storage:} Supabase Storage for technical report PDFs (user-uploaded documents)
    \item \textbf{AI/LLM:} OpenAI GPT-4 API for extraction and conversational intelligence
    \item \textbf{Web Scraping:} Firecrawl API for document extraction from regulatory sites
    \item \textbf{News:} News API for real-time mining industry announcements
    \item \textbf{API Routes:} Next.js API routes for backend logic
\end{itemize}

\subsubsection{Data Sources}
\begin{itemize}
    \item \textbf{SEDAR:} Canadian regulatory filings (System for Electronic Document Analysis and Retrieval)
    \item \textbf{SEC EDGAR:} US regulatory filings (Electronic Data Gathering, Analysis, and Retrieval)
    \item \textbf{ASX:} Australian Stock Exchange announcements and technical reports
    \item \textbf{LSE:} London Stock Exchange regulatory news service (RNS)
    \item \textbf{Company Websites:} Technical report repositories and investor relations pages
    \item \textbf{News APIs:} RSS feeds and news aggregation services
\end{itemize}

\subsubsection{Extraction Pipeline}

\textbf{Step 1: Document Acquisition}
\begin{itemize}
    \item Automated scraping of SEDAR, SEC EDGAR, ASX, LSE
    \item Firecrawl API extracts text from PDF technical reports (400--500 pages)
    \item User uploads for private/proprietary documents
\end{itemize}

\textbf{Step 2: Document Processing}
\begin{itemize}
    \item GPT-4 with structured extraction prompts
    \item Multi-pass extraction: executive summary $\rightarrow$ mineral resources $\rightarrow$ mineral reserves $\rightarrow$ economics section
    \item Entity recognition for qualified persons, companies, locations
\end{itemize}

\textbf{Step 3: Validation}
\begin{itemize}
    \item Cross-reference metrics across multiple report sections
    \item Sanity checks (e.g., NPV must be positive, IRR 5--40\%, CAPEX < \$10B)
    \item Qualified person validation against professional directories (P.Geo, P.Eng, etc.)
\end{itemize}

\textbf{Step 4: Storage}
\begin{itemize}
    \item Structured data stored in Supabase PostgreSQL
    \item Original PDFs stored in Supabase Storage with signed URLs
    \item Geospatial coordinates indexed with PostGIS extension
\end{itemize}

\section{Core Features — Detailed Specifications}

\subsection{1. Project Screener \& Database}

\textbf{Overview:} Dense, Bloomberg-style table interface displaying 15,000+ mining projects with advanced filtering, search, sorting, and bulk actions.

\subsubsection{Data Schema}

Each project in the database contains:

\begin{itemize}
    \item \textbf{Identifiers:} Project ID, Company ID, Project Name, Company Name
    \item \textbf{Location:} Country, Province/State, Latitude, Longitude
    \item \textbf{Stage:} Exploration, Pre-Feasibility, Feasibility, Development, Construction, Production, On Hold
    \item \textbf{Commodities:} Primary and secondary commodities (gold, copper, lithium, nickel, rare earths, etc.)
    \item \textbf{Financial Metrics:}
    \begin{itemize}
        \item NPV (Net Present Value in millions USD)
        \item IRR (Internal Rate of Return as percentage)
        \item CAPEX (Capital Expenditure in millions USD)
        \item AISC (All-In Sustaining Cost in USD per oz/lb/tonne)
        \item Payback Period (years)
        \item Production Rate (oz/year, tonnes/year)
    \end{itemize}
    \item \textbf{Resource Estimates:} Measured, Indicated, Inferred (tonnes @ grade)
    \item \textbf{Reserve Estimates:} Proven \& Probable (tonnes @ grade)
    \item \textbf{Qualified Persons:} Report authors with credentials (P.Geo, P.Eng, MAIG, etc.)
    \item \textbf{Technical Reports:} URLs to NI 43-101, JORC, 43-101F1 reports
    \item \textbf{Status:} Active, On Hold, Abandoned
    \item \textbf{User Fields:} Watchlist flag, created\_at, updated\_at, user\_id (for private projects)
\end{itemize}

\subsubsection{Filtering System}

\textbf{Multi-Select Filters:}
\begin{itemize}
    \item \textbf{Commodity:} Gold, Copper, Lithium, Nickel, Cobalt, Rare Earths, Uranium, Zinc, Silver, etc. (support for multiple selections)
    \item \textbf{Country/Jurisdiction:} Canada, Australia, USA, Chile, Peru, Mexico, etc.
    \item \textbf{Stage:} Exploration, Pre-Feasibility, Feasibility, Development, Construction, Production
    \item \textbf{Risk Level:} Low, Medium, High, Very High (calculated from jurisdiction, commodity volatility, legal issues)
\end{itemize}

\textbf{Range Filters:}
\begin{itemize}
    \item \textbf{NPV Range:} Slider from \$0M to \$10,000M with custom input fields
    \item \textbf{IRR Range:} Slider from 0\% to 50\% with custom input fields
    \item \textbf{CAPEX Range:} Slider from \$0M to \$5,000M with custom input fields
    \item \textbf{AISC Range:} Slider from \$0/oz to \$2,000/oz with custom input fields
\end{itemize}

\textbf{Search:}
\begin{itemize}
    \item Global search across project names, company names, locations
    \item Real-time filtering as user types
    \item Fuzzy matching for misspellings
\end{itemize}

\subsubsection{Table Features}

\begin{itemize}
    \item \textbf{Column Management:} Show/hide columns via dropdown menu
    \item \textbf{Sortable Headers:} Click to sort by any column (ascending/descending)
    \item \textbf{Row Selection:} Checkbox column for multi-select (supports Shift+Click for range selection)
    \item \textbf{Pagination:} 25/50/100/200 rows per page with page navigation
    \item \textbf{Conditional Formatting:}
    \begin{itemize}

        \item Red highlight for IRR < 10\%

\subsubsection{Bulk Actions}

\begin{itemize}

\subsection{6. Mining Agent — Automated Data Collection}

\textbf{Overview:} Real-time scraping and extraction orchestration system for ingesting new technical reports and regulatory filings.

\subsubsection{Architecture}

\textbf{Components:}
\begin{enumerate}
    \item \textbf{Scrapers:} SEDAR scraper, SEC EDGAR scraper, ASX scraper, LSE scraper, generic web scraper
    \item \textbf{Document Processor:} Firecrawl API for PDF text extraction
    \item \textbf{LLM Extractor:} GPT-4-powered structured data extraction
    \item \textbf{Validator:} Cross-references metrics, sanity checks, qualified person verification
    \item \textbf{Database Writer:} Stores extracted data in Supabase PostgreSQL
    \item \textbf{Progress Tracker:} Real-time updates via WebSocket for UI progress bar
\end{enumerate}

\textbf{Workflow:}
\begin{enumerate}
    \item User clicks "Run Mining Agent" or agent runs on schedule (daily cron job)
    \item Agent fetches new filings from SEDAR, SEC EDGAR, ASX, LSE (based on last run timestamp)
    \item Identifies NI 43-101, JORC, PEA, PFS, DFS filings by keyword matching
    \item Downloads PDFs, sends to Firecrawl API for text extraction
    \item Sends extracted text to GPT-4 with structured prompts (extract NPV, IRR, CAPEX, reserves)
    \item Validates extracted data (sanity checks, cross-references)
    \item Writes to database, updates projects table
    \item Sends progress updates to UI via WebSocket ("Processing 10/50 documents...")
    \item On completion, triggers refresh of screener table
\end{enumerate}

\subsubsection{User Upload}

\begin{itemize}
    \item \textbf{Drag-and-Drop:} Upload PDF technical reports from file system
    \item \textbf{Private Projects:} User-uploaded projects are private by default (user\_id field, RLS policy)
    \item \textbf{Automatic Extraction:} Same LLM pipeline extracts NPV, IRR, CAPEX, reserves
    \item \textbf{Secure Storage:} PDFs stored in Supabase Storage with user-level access control
    \item \textbf{Full Feature Access:} Private projects appear in screener (only for owner), support AI chat, comparison, sensitivity
\end{itemize}
